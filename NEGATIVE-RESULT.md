## 3. Why the carry didn't lift (root cause analysis)

Three orthogonal reasons, all documented from the on-disk evidence:

### 3.1 Carry fires too rarely to be an income stream

The Binance funding-rate distribution for the backtest window (2024-01 → 2026-07, 8h events) shows the carry threshold (|funding| > 0.01% per 8h) is rarely crossed:

| Symbol | Bars (8h events) | Positive% | Negative% | Neutral% |
|---|---:|---:|---:|---:|
| BTC | 7,466 | 14.3% | 2.5% | **83.1%** |
| ETH | 7,232 | 17.0% | 2.3% | **80.7%** |
| SOL | 6,433 | 13.0% | 11.8% | **75.2%** |

Source: `data/funding/binance_*usdt_funding_8h.csv` (REAL Binance data, not synthetic — verified via Track C stdout `funding-rate carry engaged; mode=2of3; bars=N; funding-distribution=...`). 75-83% of bars sit in the neutral band, where the carry abstains and the wrapped DonchianPivot signal passes through unchanged. This means the carry can only modulate the signal 17-25% of the time at most.

### 3.2 Where it does fire, the signal is anti-correlated with the existing edge

When SOL's funding rate is positive (13.0% of bars, 8h periods when shorts pay longs), the carry says "short bias". But SOL's Donchian+Pivot edge in this regime is LONG-biased (Phase 19 #1 at cap=0.12: SOL monthlyReturn=37.91%/mo, 10576 trades, winrate=68.21%). The carry is fighting the existing edge on the highest-return symbol.

### 3.3 Edge-INVARIANCE test confirms carry is NOT a winning-trade filter

The Phase 20/21 archive mandated an edge-INVARIANCE pre-flight: split backtest by funding-rate sign (positive vs negative vs neutral), compare win-rate per bucket. The result (from Track A's deliverable, see `deliverable.md`): win-rate spread across buckets was **< 5pp** for all three symbols. The carry is not a winning-trade filter — it does not predict which trades will win. Per the brief's Note clause: "If win-rate spread is < 5pp, the funding-rate classifier is not a winning-trade filter, but the carry is still an INCOME stream (not a filter) — track can still PASS as long as Track C reports the +2-5 pp/mo lift from the income stream."

Track C does NOT report a +2-5 pp/mo lift. It reports **−0.52pp**. Track A's edge-INVARIANCE pre-flight correctly flagged this risk; Track C's empirical sweep confirmed it.
