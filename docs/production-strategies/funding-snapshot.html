Funding pillanatkép típus (FundingSnapshot)

Mi ez?

Ez a legegyszerűbb fájl a kódbázisban: egy típusdefiníció. Nincs benne logika, nincs benne kereskedés — csak azt írja le, hogyan néz ki egy funding kamat adat egy adott pillanatban.

Az egész fájl mindössze 25 sor, és ez áll benne:

export interface FundingSnapshot {
  readonly fundingTime: number;     // mikor fizették (Unix timestamp ms-ban)
  readonly symbol: string;          // melyik coin (pl. "BTC-USD")
  readonly fundingRate: number;      // a kamat értéke (pl. 0.0001 = 0.01%)
  readonly markPrice?: number;       // opcionális: a jegyzési ár
}

Mire használja a rendszer?

A dYdX kontra CEX carry stratégia és a paper-trade runner használja. Minden 8 órában kapnak egy új FundingSnapshot-ot a dYdX-ről és egy másikat a CEX-ről. A kettőt összehasonlítva döntik el, hogy érdemes-e carry pozíciót nyitni.

Példa: ha a dYdX FundingSnapshot.fundingRate = -0.0001 (long kap pénzt), és a CEX FundingSnapshot.fundingRate = +0.0003 (short kap pénzt), akkor long-dYdX + short-CEX carry-t nyit a rendszer.

Miért van külön fájlban?

Korábban ez a FundingSnapshot típus egy nagyobb fájlban volt (funding-carry.ts), amit a Phase 32 cleanup során töröltünk. Mivel a dYdX carry stratégia továbbra is használja ezt a típust, kiemeltük egy külön fájlba, hogy a típus önállóan is fenntartható legyen.

Ha bármikor újra szükség lenne a teljes funding-carry stratégiára, a FundingSnapshot típus már itt van, és a törölt funding-carry.ts tartalma elérhető a git history-ban.
